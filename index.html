面向对象高级（二）

01 多态、使用多态的好处

  • 多态是在继承/实现情况下的一种现象，表现为：对象多态、行为多态
  • 具体代码体现

  • 多态的前提：有继承/实现关系；存在父类引用子类对象；存在方法重写

  • 注意事项：多态是对象、行为的多态，java中的成员变量不谈多态

  • 使用多态的好处

    • 在多态形式下，右边对象是解耦合的，便于扩展和维护
    • 定义方法时，使用父类类型的形参，可以接收一球子类对象，扩展性更强、更便利

  • 类型转换

    • 自动类型转换：父类 变量名=new子类（）；

    • 强制类型转换：子类 变量名=new（子类）父类变量；

    • 注意事项

      • 存在继承/实现关系就可以在编译阶段进行强制类型转换，编译阶段不会报错
      • 运行时，如果发现对象的真实类型与强转后的类型不同，就会报类型转换异常（ClassCastException）的错误出来

    • 强转时，java建议：使用instanceof，判断当前对象的真实类型

02 final、常量

  • final

    • final是最终的意思，可以修饰类，方法，变量
    • 修饰类：该类称为最终类，不可以被继承
    • 修饰方法：该方法为最终方法，不可以被重写
    • 修饰变量：该变量只能被赋值一次

  • final修饰变量注意事项

    • final修饰基本类型的变量，变量存储的数据不能被改变
    • final修饰引用类型的变量，变量存储的地址不能被改变

  • 常量

    • 使用了static final修饰的成员变量称为常量
    • 作用：通常用于记录系统的配置信息

  • 使用常量记录系统配置信息的优势、执行原理

    • 代码可读性更好，可维护性也更好
    • 程序编译后，常量会被“宏替换”：出现常量的地方全部会被替换成其记住的字面量，这样可以保证使用常量和直接使用字面量的性能是一样的

03 抽象类-认识抽象类和其好处

  • 抽象类和抽象方法都是用abstrac修饰的；抽象方法只有方法签名，不能写方法体
  • 抽象类的注意事项和特点
    • 抽象类中可以不写抽象方法，但又抽象方法的类一定是抽象类
    • 类有的成员（成员变量、方法、构造器），抽象类都具备
    • 抽象类不能创建对象，仅作为一种特殊的父类，让子类继承并实现
    • 一个类继承抽象类，必须重写完抽象类的所有方法，否则这个类也必须定义为一个抽象类
  • 抽象类的应用场景和好处
    • 父类知道每一个子类都要做某个行为，但每个子类要做的情况不一样，父类就定义成抽象方法，交给子类去重写实现
    • 我们抽出这样的抽象类，是为了更好的支持多态

04 抽象类的应用-模板方法设计模式

  • 模板方法设计模式解决了方法中存在重复代码的问题
  • 模板方法设计模式应该怎么写
    • 定义一个抽象类
    • 在抽象类李定义两个方法
      • 一个是模板方法：放相同的代码
      • 一个是抽象方法：具体实现交给子类完成
  • 模板方法建议使用final关键字修饰

05 接口：认识接口，使用接口的好处

  • java提供了一个关键字interface，用这个关键字我们可以定义出一个特殊的结构：接口
  • 注意：接口不能创建对象；接口是用来被类实现(implements)的，实现接口的类称为实现类

  • 一个类可以实现多个接口（接口可以理解为干爹），实现类实现多个接口，必须重写完所有接口的全部抽象方法，否则实现类需要定义成抽象类

  • 接口的好处（重点）

    • 弥补了类单继承的不足，一个类同时可以实现多个接口
    • 让程序可以面向接口编程，这样程序员就可以灵活方便地求换各种业务实现
    • 使用接口去找干爹，别人通过你implements的接口，就可以显性的知道你是谁，从而也就可以放心的把你当作谁来用了
    • 一个类我们可以实现多个接口，同样，一个接口也可以被多个类实现。这样做的好处是我们的程序就可以面向接口编程了，这样可以很方便地灵活切换各种业务实现了

07 接口：JDK8开始接口新增的方法、接口的多继承、注意事项

  • JDK8开始，接口新增了哪些方法
    • 默认方法：default修饰，使用实现类的对象调用
    • 静态方法：static修饰，必须用当前接口名调用
    • 私有方法：private修饰，jdk9开始，只能在接口内被调用
    • 默认他们都会被public修饰
  • JDK8开始，新增的方法增强了接口的能力，更便于项目的扩展和维护

面向对象高级（一）

01 static修饰成员变量，类变量的应用场景

  • static叫静态，可以修饰成员变量、成员方法

  • 成员变量按照有无static修饰，分为两种：

    • 类变量：有static修饰，属于类，在计算机里只有一份，会被类的全部对象共享

      • 使用
      • 属于类，与类一起加载一次，在内存中只有一份，会被类的所有对象共享

    • 实例变量（对象的变量）：无static修饰，属于每个对象的

      • 使用
      • 属于对象，每个对象中都有一份

  • 访问自己类中的类变量是，可以省略类名不写

02 static修饰成员方法

  • 成员方法按照有无static修饰，分为两类：

    • 类方法（静态方法）

      • 使用
      • 属于类，可以直接类名访问，也可以用对象访问

    • 实例方法（对象的方法）

      • 使用
      • 属于对象，只能用对象访问

  • main方法的解析

java Test –> Test.main(…)

03 static修饰成员方法的应用场景-工具类

  • 类方法的应用场景，可以用来设计工具类
  • 工具类
    • 工具类中的方法都是类方法，每个类方法都是用来完成一个功能的
    • 提高了代码的复用性；调用方便，提高了开发效率
  • 使用类方法不需要调用对象，避免内存浪费
  • 工具类不需要创建对象，建议将构造器私有化

04 static的注意事项

  • 类方法中可以直接访问类的成员，不可以直接访问实例成员
  • 实例方法中既可以直接访问类成员，也可以直接访问实例成员
  • 实例方法中可以出现this关键字，类方法中不可以出现this关键字的

05 static应用-代码块

  • 代码块概述

代码块是类的五大成分之一（成员变量，构造器，方法，构造器，内部类）

  • 代码块分类
    • 静态代码块
      • 格式：static{}
      • 特点：类加载是自动执行，由于类只会加载一次，所以静态代码块也只会执行一次
      • 作用：完成类的初始化，例如对类变量的初始化赋值
    • 实例代码块
      • 格式：{}
      • 特点：每次创建对象时，执行实例代码块，并在构造器前执行
      • 作用：和构造器一样，都是用来完成对象的初始化的，例如：对实例变量进行初始化赋值

06 static应用-单例设计模式，饿汉式单例，懒汉式单例

  • 设计模式就是具体问题的最优解决方案
  • 单例模式确保一个类只有一个对象
  • 单例特点
    • 类的构造器私有化
    • 定义一个类变量存储类的一个对象
    • 提供一个类方法返回对象
  • 单例的好处
    • 如任务管理器，使用单例避免浪费内存
  • 饿汉式单例：在获取类的对象时，对象已经创建好了
  • 懒汉式单例：拿对象时，才开始创建对象

07 继承：使用继承的好处

  • 继承就是用extends关键字，让一个类和另一个类建立起一种父子关系
  • 子类可以继承父类非私有的成员
  • 带继承关系的类，java会用类和其父类一起创建类的对象
  • 对象能直接访问的成员有父子类共同决定

08 继承：权限修饰符

  • 权限修饰符用来限制类中的成员（成员变量，成员方法，构造器，代码块…）能够被访问的范围
  • 分类
    • private：最大到 在本类中
    • 缺省：最大到 同一个包下的其他类里
    • protected：最大到 任意包下的子类里
    • public：最大到 任意包下的任意类里

09 继承：单继承，Object，方法重写

  • java时单继承的：不支持多继承，但支持多层继承
  • Object类时Java中所有类的祖宗
  • 方法重写
    • 定义：子类写了一个方法名称，形参列表和父类某个方法一样的方法去覆盖父类的该方法
    • 注意事项：
      • 建议加上：@Override注解，检查且可读性好
      • 子类重写父类方法时，访问权限必须大于或者等于父类被重写的方法的权限
      • 重写的方法返回值类型，必须与被重写方法的返回值类型一样，或者范围更小
      • 私有方法、静态方法不能被重写
    • 应用场景：当父类方法不满足子类需求时可以重写

10 继承：子类访问成员的特点

  • 访问依照就近原则
    • 先子类局部范围找
    • 然后子类成员范围找
    • 然后父类成员范围找，如果父类范围还没有找到就报错
  • 强制使用父类：super.成员变量/成员方法

11 继承：子类构造器的特点，super，this调用兄弟构造器

  • 子类构造器的特点：先调用父类构造器再执行自己的
  • 子类构造器是如何实现调用父类构造器的
    • 默认情况下，子类全部构造器的第一行代码都是super（），会调用父类的无参构造器
    • 如果父类没有无参构造器，我们必须再子类构造器第一行手写其无参构造器
  • 调用父类有参构造器的场景：位对象中包含父类这部分的成员变量进行赋值
  • this（…）用于在构造器中调用本类的其他构造器
  • this和super都需要放在构造器第一行

Java方法

01 方法概述

  • 方法是一种语法结构，它可以把一段代码封装成一个功能，以便重复调用

  • 完整格式

  • 方法执行必须调用

  • 可以提高代码复用性，提高开发效率，程序逻辑清晰

02 方法定义的其他形式，常见问题

  • 不需要接收数据，可以不写形参列表
  • 不需要返回值，申明返回类型为void
  • 常见问题
    • 方法在类中的位置无所谓，但是不能方法中定义方法
    • void类型的方法不能return数据，其他类型的方法必须return数据
    • return下不能写代码
    • 不调用不执行，要调用需要严格匹配参数情况
    • 调用有返回值的方法
      • 定义变量接收结果
      • 直接输出调用
      • 直接调用
    • 调用无返回值的方法
      • 只能直接调用

03 方法的案例讲解

  • 关注是否接收数据，关注是否返回数据

04 方法在计算机中的执行

  • 方法运行区域在栈内存
  • 先进后出是栈的特点
  • 使用栈，保证一个方法调用完另一个方法后可以回来

05 方法参数传递

  • 值传递：传输的是实参存储的值的副本
  • 实参：在方法内部定义的变量
  • 形参：以方法为例，就是方法定义时的变量
  • 基本类型和引用类型在参数传递时的异同
    • 都是值传递
    • 基本类型的参数传递是数据值，引用类型的参数传递是地址值

06 参数传递的案例

07 方法重载，return关键字

  • 一个类中，多个方法的名称相同，形参列表不同，此时方法重载
  • 注意事项
    • 一个类中，只要一些方法名称相同，形参列表不同，那么就是重载了
    • 形参列表不同：形参的个数，类型，顺序不同，不关心形参的名称
  • 应用场景：需要为一类业务提供多种解决方案
  • return；跳出并立即结束所在方法的执行
  • break；跳出并技术当前所在循环的执行
  • continue：结束当前所在循环的当次继续，进入下一次执行
