P1244

tag：dp，递推

解：

思路

设dp[i] [j]表示i片荷叶j个石墩的答案

石墩联想到汉诺塔，但要理解好荷叶的作用

由特殊到一般，考虑以石墩为序：

  • dp[i] [0]表示i片荷叶0个石墩，显然答案是i+1。进一步，发现荷叶仅仅起中转作用；
  • dp[i] [1]，用1个石墩来暂存，最多暂存dp[i] [0]个，故答案为dp[i] [0]+（i+1）；
  • dp[i] [2]，此时有2个石墩来暂存，最多暂存dp[i] [1]+dp[i] [0]个，故答案为dp[i] [1]+dp[i] [0]+（i+1）；

规律已出

  • 边界：dp[i] [0]=i+1；
  • 转移方程：dp[i] [j]=dp[i] [j-1]+（i+1）；

代码

反思：

  • 分析复杂题意，抽象为熟悉模型
  • 关注每一种元素的实际作用和性质，如荷叶只是暂存1个，石墩可以由子问题简单相加构成而且也只是暂存

P1025

tag：搜索，递推，剪枝

解：

解法一(dfs)：

思路

将答案指定为升序，依此遍历解

注意在进一步搜索时可以通过题意限制剪枝，如 和固定

code

解法二(dp)：

思路

设dp[i] [j]:将i划分为j份的方案数

  • 边界：dp[i] [1]=i；
  • 转移方程：
    • 当前方案含有1:dp[i-1] [j-1]–>任意i划分为j份的含1的方案 都可以被一一映射为 i-1划分为j-1份的方案+{1}
    • 当前方案不含1:dp[i-j] [j]–>任意i划分为j分的不含1的方案 都可以被一一映射为 i-j划分为j份的方案+{1,1,…,1}

code

反思：

  • 先用暴搜，尝试剪枝
  • 转移复杂时考虑分类简化问题，用映射的思想来转移
