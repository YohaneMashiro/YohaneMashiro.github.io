Git

01 学习目标

  • 1.记录历史版本，回退历史版本
  • 2.团队开发，方便代码合并

02 版本控制器的方式

  • SVN：集中式版本控制系统，版本库位于中央服务器
  • Git：分布式版本控制系统，分为本地仓库和远程仓库

03 SVN_Git

  • SVN为集中版本控制工具，不支持离线提交
  • Git为分布式版本控制工具，支持离线提交

04 git工作流程简述

  • 本地仓库（Repository）
    • 工作区（Workspace）
    • 暂存区（IndexStage）
  • 远程仓库（Remote）

11 分支使用流程

  • master
    • 实际运行版本
  • develop
    • 开发版本，修改后merge到master
  • feature
    • 开发具体功能版本，修改后merge到develop，用完detele
  • hotfix
    • 修复master的bug后版本，merge到master并创建一个develop版本
  • else

21 远程解决冲突

  • 多人同时修改了并push，和本地冲突解决方案一样
    • 1.pull得到文件，发现冲突
    • 2.解决冲突，文件结果自定
    • 3.解决完成push
    • 4.对方及时pull更新

23 idea配置，本地基本操作_push

  • file–>setting–>git–>Path to Git exe.
  • 创建远程仓库
  • 注意在工程里添加.gitignore文件
  • 本地仓库:VCS–>Import into Version Control–>Create Git Repository
  • commit:Git–>✔️–>选文件，写log
  • 查看:Version Control–>Log
  • push:VCS–>Git–>Push–>Define remote–>URL;以后可以在commit时一起push了

24 idea克隆，更新，解决冲突

  • VCS–>Checkout from Version Control–>Git–>URL&Dir
  • 冲突
    • A commit&push
    • B commit
    • B pull(↙️),发现冲突，同理解决冲突
    • B 右键文件:Git–>Add;commit&push
