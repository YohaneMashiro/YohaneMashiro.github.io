面向对象高级（一）

01 static修饰成员变量，类变量的应用场景

  • static叫静态，可以修饰成员变量、成员方法

  • 成员变量按照有无static修饰，分为两种：

    • 类变量：有static修饰，属于类，在计算机里只有一份，会被类的全部对象共享

      • 使用
      • 属于类，与类一起加载一次，在内存中只有一份，会被类的所有对象共享

    • 实例变量（对象的变量）：无static修饰，属于每个对象的

      • 使用
      • 属于对象，每个对象中都有一份

  • 访问自己类中的类变量是，可以省略类名不写

02 static修饰成员方法

  • 成员方法按照有无static修饰，分为两类：

    • 类方法（静态方法）

      • 使用
      • 属于类，可以直接类名访问，也可以用对象访问

    • 实例方法（对象的方法）

      • 使用
      • 属于对象，只能用对象访问

  • main方法的解析

java Test –> Test.main(…)

03 static修饰成员方法的应用场景-工具类

  • 类方法的应用场景，可以用来设计工具类
  • 工具类
    • 工具类中的方法都是类方法，每个类方法都是用来完成一个功能的
    • 提高了代码的复用性；调用方便，提高了开发效率
  • 使用类方法不需要调用对象，避免内存浪费
  • 工具类不需要创建对象，建议将构造器私有化

04 static的注意事项

  • 类方法中可以直接访问类的成员，不可以直接访问实例成员
  • 实例方法中既可以直接访问类成员，也可以直接访问实例成员
  • 实例方法中可以出现this关键字，类方法中不可以出现this关键字的

05 static应用-代码块

  • 代码块概述

代码块是类的五大成分之一（成员变量，构造器，方法，构造器，内部类）

  • 代码块分类
    • 静态代码块
      • 格式：static{}
      • 特点：类加载是自动执行，由于类只会加载一次，所以静态代码块也只会执行一次
      • 作用：完成类的初始化，例如对类变量的初始化赋值
    • 实例代码块
      • 格式：{}
      • 特点：每次创建对象时，执行实例代码块，并在构造器前执行
      • 作用：和构造器一样，都是用来完成对象的初始化的，例如：对实例变量进行初始化赋值

06 static应用-单例设计模式，饿汉式单例，懒汉式单例

  • 设计模式就是具体问题的最优解决方案
  • 单例模式确保一个类只有一个对象
  • 单例特点
    • 类的构造器私有化
    • 定义一个类变量存储类的一个对象
    • 提供一个类方法返回对象
  • 单例的好处
    • 如任务管理器，使用单例避免浪费内存
  • 饿汉式单例：在获取类的对象时，对象已经创建好了
  • 懒汉式单例：拿对象时，才开始创建对象

07 继承：使用继承的好处

  • 继承就是用extends关键字，让一个类和另一个类建立起一种父子关系
  • 子类可以继承父类非私有的成员
  • 带继承关系的类，java会用类和其父类一起创建类的对象
  • 对象能直接访问的成员有父子类共同决定

08 继承：权限修饰符

  • 权限修饰符用来限制类中的成员（成员变量，成员方法，构造器，代码块…）能够被访问的范围
  • 分类
    • private：最大到 在本类中
    • 缺省：最大到 同一个包下的其他类里
    • protected：最大到 任意包下的子类里
    • public：最大到 任意包下的任意类里

09 继承：单继承，Object，方法重写

  • java时单继承的：不支持多继承，但支持多层继承
  • Object类时Java中所有类的祖宗
  • 方法重写
    • 定义：子类写了一个方法名称，形参列表和父类某个方法一样的方法去覆盖父类的该方法
    • 注意事项：
      • 建议加上：@Override注解，检查且可读性好
      • 子类重写父类方法时，访问权限必须大于或者等于父类被重写的方法的权限
      • 重写的方法返回值类型，必须与被重写方法的返回值类型一样，或者范围更小
      • 私有方法、静态方法不能被重写
    • 应用场景：当父类方法不满足子类需求时可以重写

10 继承：子类访问成员的特点

  • 访问依照就近原则
    • 先子类局部范围找
    • 然后子类成员范围找
    • 然后父类成员范围找，如果父类范围还没有找到就报错
  • 强制使用父类：super.成员变量/成员方法

11 继承：子类构造器的特点，super，this调用兄弟构造器

  • 子类构造器的特点：先调用父类构造器再执行自己的
  • 子类构造器是如何实现调用父类构造器的
    • 默认情况下，子类全部构造器的第一行代码都是super（），会调用父类的无参构造器
    • 如果父类没有无参构造器，我们必须再子类构造器第一行手写其无参构造器
  • 调用父类有参构造器的场景：位对象中包含父类这部分的成员变量进行赋值
  • this（…）用于在构造器中调用本类的其他构造器
  • this和super都需要放在构造器第一行
